# High-Tech Personal Profile Website Blueprint

## Project Goal

Create a personal profile website that feels like a **premium technology showcase** while still staying professional, readable, and accessible. The website should present your current IT and computer science background first, while also showing your customer service experience, education, transcript highlights, classes, projects, and downloadable resume.

This website should make visitors feel that they are looking at someone who understands:

- modern technology
- professional presentation
- strong UI/UX thinking
- accessibility and ADA-friendly design
- polished front-end detail
- animation and interaction design
- real-world IT experience

The final feeling should be:

- futuristic
- modern
- impressive
- clean
- highly visual
- interactive
- professional
- memorable

The website should not feel overloaded or flashy in a bad way. It should feel **intentional**, **high-end**, and **well-designed**.

---

## Main Website Strategy

The website should do two things at the same time:

1. **Act like a professional portfolio**
2. **Act like a visual technology showcase**

That means the site should not only tell people about your background. It should also visually prove that you understand design, usability, interaction, and modern web presentation.

The site should focus mostly on:

- IT Administrator experience
- Computer Science background
- Web and technical skills
- Cloud and systems interest
- Real projects
- Customer service strength
- Professional growth

Your hotel background should still be included because it shows:

- communication
- professionalism
- leadership
- customer support
- conflict resolution
- people skills

That combination is powerful because it shows both **technical ability** and **human ability**.

# ADA / Accessibility Requirements

## This Must Be Taken Seriously

A beautiful site is not enough. It must also be accessible.

## Key Accessibility Requirements

### 1. Color Contrast
Text must have enough contrast against backgrounds in both dark and light mode.

Avoid:
- light gray text on white
- low-contrast neon text
- faded labels

### 2. Proper Heading Structure
Use headings in order:
- H1 for main page title
- H2 for major sections
- H3 for subsections

Do not skip heading levels randomly.

### 3. Alt Text
All important images must have meaningful alt text.

Examples:
- “Professional portrait of Hlay Aliotte”
- “Screenshot of volunteer web application homepage”
- “MQS HVAC website project preview”

Decorative graphics should have empty alt values so screen readers ignore them.

### 4. Keyboard Navigation
Everything must be usable with keyboard:
- menu
- buttons
- form fields
- toggles
- modals

Focus states must be visible.

### 5. Link Text
Use clear links like:
- Download Resume
- View Project Details
- Contact Me

Avoid vague text like:
- Click here
- Read more

### 6. Forms
Every form field must have:
- visible label
- accessible input
- error message
- helpful instructions if needed

### 7. Motion Accessibility
Provide reduced motion support.

If user prefers reduced motion:
- limit large transitions
- reduce parallax
- reduce animation intensity
- keep interface comfortable

### 8. Responsive Readability
Text should remain readable on all devices.

### 9. Screen Reader Friendly Structure
Use semantic HTML:
- header
- nav
- main
- section
- article
- footer

### 10. Dark and Light Mode Accessibility
Both themes must remain readable and accessible.

## Why It Matters

This feature makes the site more modern and more enjoyable for users.

## Best Practice
- detect system preference on first load
- allow manual toggle
- save user preference locally
- ensure both themes look polished

## Toggle Style
The toggle should feel premium.

Ideas:
- moon/sun icon switch
- animated sliding pill toggle
- soft glow effect when switching
- quick but smooth transition

## Theme Rules
Do not simply invert colors.

Design each theme intentionally:
- background
- text
- cards
- borders
- shadows
- hover states
- glow effects

## ADA Reminder
Both themes must keep strong contrast and readability.
